# Deskripsi Program

Program ini merupakan implementasi sederhana dari fungsi penjumlahan yang dilengkapi dengan validasi input dan unit testing menggunakan fitur bawaan Node.js Test Runner.

Tujuan program adalah:

* Melakukan operasi penjumlahan dua angka.
* Memastikan input yang diberikan bertipe number.
* Menghasilkan error apabila input tidak valid.
* Melakukan pengujian otomatis menggunakan Unit Testing.

# Source Code

## File: hitung.js

```javascript
function tambahPengitung(terkini, jumlah) {

    // validasi input harus angka
    if (typeof terkini !== "number" || typeof jumlah !== "number") {
        throw new Error("Input harus berupa angka");
    }

    return terkini + jumlah;
}

export { tambahPengitung };
```

### Penjelasan

Fungsi `tambahPengitung()` menerima dua parameter:

| Parameter | Keterangan                  |
| --------- | --------------------------- |
| terkini   | Nilai awal                  |
| jumlah    | Nilai yang akan ditambahkan |

Sebelum melakukan operasi penjumlahan, program memvalidasi apakah kedua parameter bertipe number.

Jika bukan angka, maka program akan menghasilkan exception:

```javascript
throw new Error("Input harus berupa angka");
```

## File: hitung.test.js

```javascript
import test from 'node:test';
import assert from 'node:assert';
import { tambahPengitung } from './hitung.js';

test('5 tambah 3 sama dengan 8', () => {
    assert.strictEqual(tambahPengitung(5, 3), 8);
});

test('0 tambah 10 sama dengan 10', () => {
    assert.strictEqual(tambahPengitung(0, 10), 10);
});

test('angka negatif', () => {
    assert.strictEqual(tambahPengitung(-5, 2), -3);
});

test('harus error jika bukan angka', () => {
    assert.throws(() => {
        tambahPengitung("5", 3);
    });
});
```

# Analisis Debugging

Pada fungsi utama terdapat validasi:

```javascript
if (typeof terkini !== "number" || typeof jumlah !== "number") {
    throw new Error("Input harus berupa angka");
}
```

Validasi ini mencegah:

* String
* Boolean
* Array
* Object
* Null

untuk diproses sebagai angka.

Contoh:

```javascript
tambahPengitung("5", 3);
```

akan menghasilkan:

```text
Error: Input harus berupa angka
```

Hal ini membantu proses debugging karena kesalahan dapat terdeteksi lebih awal.

# Analisis Clean Code

## 1. Meaningful Naming

```javascript
tambahPengitung()
```

Nama fungsi menjelaskan tujuan fungsi secara langsung.

---

## 2. Single Responsibility Principle

Fungsi hanya memiliki satu tugas:

```text
Menjumlahkan dua angka
```

Tidak melakukan input/output ataupun manipulasi data lainnya.

---

## 3. Readability

Kode mudah dipahami karena:

* Struktur sederhana.
* Indentasi konsisten.
* Variabel memiliki nama yang jelas.
* Komentar seperlunya.

---

## 4. Defensive Programming

Validasi input dilakukan sebelum proses utama dijalankan.

```javascript
if (typeof terkini !== "number")
```

Teknik ini mengurangi potensi bug pada runtime.

---

# Kesimpulan

Program berhasil menerapkan Unit Testing menggunakan Node.js Test Runner untuk memverifikasi fungsi penjumlahan. Empat skenario pengujian berhasil dijalankan dengan hasil 100% lulus. Selain itu, validasi input diterapkan untuk meningkatkan keandalan program dan mempermudah proses debugging. Dari sisi Clean Code, program telah menerapkan prinsip Single Responsibility Principle, Meaningful Naming, Readability, dan Defensive Programming sehingga kode lebih mudah dipahami, diuji, dan dipelihara.
