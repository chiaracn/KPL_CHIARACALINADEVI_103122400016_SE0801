### Event Delegation dalam Design Pattern JavaScript

**Event Delegation** adalah teknik dalam JavaScript yang memanfaatkan mekanisme **event bubbling** untuk menangani event dari banyak elemen melalui satu event listener yang ditempatkan pada elemen induk (parent). Dengan kata lain, kita tidak perlu memasang event listener pada setiap elemen secara langsung, tetapi cukup pada parent yang membungkus elemen-elemen tersebut.

#### Cara Kerja Event Delegation

Saat sebuah event terjadi pada suatu elemen (misalnya tombol diklik), event tersebut akan berjalan dari elemen target menuju elemen-elemen induknya. Proses ini disebut **event bubbling**. Event delegation memanfaatkan proses ini dengan cara menangkap event pada parent, lalu memeriksa elemen mana yang sebenarnya memicu event tersebut.

#### Contoh Tanpa Event Delegation

```javascript
const buttons = document.querySelectorAll(".delete-btn");

buttons.forEach(button => {
    button.addEventListener("click", () => {
        console.log("Data dihapus");
    });
});
```

Pada contoh di atas, setiap tombol harus memiliki event listener sendiri. Jika jumlah tombol sangat banyak atau tombol ditambahkan secara dinamis, pendekatan ini kurang efisien.

#### Contoh Dengan Event Delegation

```javascript
document.getElementById("list").addEventListener("click", function(event) {
    if (event.target.classList.contains("delete-btn")) {
        console.log("Data dihapus");
    }
});
```

```html
<ul id="list">
    <li><button class="delete-btn">Hapus</button></li>
    <li><button class="delete-btn">Hapus</button></li>
    <li><button class="delete-btn">Hapus</button></li>
</ul>
```

Di sini hanya ada **satu event listener** pada elemen `<ul>`, tetapi semua tombol di dalamnya tetap dapat merespons klik.

---

### Keuntungan Event Delegation

#### 1. Lebih Efisien

Karena hanya menggunakan satu event listener, penggunaan memori menjadi lebih kecil dibandingkan memasang listener pada banyak elemen.

#### 2. Mendukung Elemen Dinamis

Jika elemen baru ditambahkan ke dalam DOM melalui JavaScript, elemen tersebut otomatis dapat merespons event tanpa perlu menambahkan listener baru.

Contoh:

```javascript
const newButton = document.createElement("button");
newButton.className = "delete-btn";
newButton.textContent = "Hapus Baru";

document.getElementById("list").appendChild(newButton);
```

Tombol baru tersebut tetap bisa diklik walaupun tidak pernah diberi event listener secara langsung.

#### 3. Kode Lebih Mudah Dipelajari dan Dipelihara

Logika penanganan event terpusat pada satu tempat sehingga kode menjadi lebih bersih (clean code) dan mudah dikelola.

---

### Hubungan dengan Design Pattern

Dalam konteks design pattern JavaScript, Event Delegation dapat dianggap sebagai pola yang mendukung **behavioral design**, karena mengatur bagaimana objek atau elemen berinteraksi melalui event. Konsep ini banyak digunakan pada framework modern seperti React, Vue, dan Angular untuk meningkatkan performa aplikasi.

Contohnya pada aplikasi e-commerce yang memiliki banyak tombol:

* Tombol tambah ke keranjang
* Tombol hapus produk
* Tombol favorit

Daripada membuat event listener untuk setiap tombol, developer dapat memasang satu listener pada container utama lalu menentukan aksi berdasarkan elemen yang diklik.

---

### Kapan Sebaiknya Menggunakan Event Delegation?

Event Delegation sangat cocok digunakan ketika:

* Jumlah elemen sangat banyak.
* Elemen sering ditambahkan atau dihapus secara dinamis.
* Ingin mengurangi penggunaan memori dan meningkatkan performa.
* Ingin membuat kode lebih sederhana dan mudah dipelihara.

Sebaliknya, jika hanya ada satu atau dua elemen yang tidak berubah, penggunaan event listener langsung pada elemen tersebut biasanya sudah cukup.

---

### Kesimpulan

**Event Delegation** adalah teknik penting dalam JavaScript yang memanfaatkan **event bubbling** untuk menangani event dari banyak elemen melalui satu event listener pada parent. Teknik ini membuat aplikasi lebih efisien, mendukung elemen yang dibuat secara dinamis, mengurangi penggunaan memori, serta menghasilkan kode yang lebih rapi dan mudah dirawat. Oleh karena itu, Event Delegation menjadi salah satu praktik yang sering digunakan dalam pengembangan aplikasi web modern dan dianggap sebagai pola desain yang efektif dalam pengelolaan event pada JavaScript.
