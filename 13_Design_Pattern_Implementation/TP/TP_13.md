Deskripsi  
Pada project ini terdapat implementasi Context Provider Pattern menggunakan React Context API pada file CartContext.jsx.
Pattern ini digunakan untuk mengelola state keranjang belanja secara global. Jadi data cart dapat dipakai oleh banyak komponen seperti halaman cart, checkout, navbar, dan halaman produk tanpa harus mengirim data melalui props satu per satu.
Design ini membuat kode lebih rapi, reusable, dan mudah dikembangkan karena semua logika cart dipusatkan pada satu tempat.
Design pattern yang paling jelas digunakan adalah: Context Provider Pattern

Pattern ini berada di:
```
frontend/src/context/CartContext.jsx
```

Controller  
Bagian Controller berada di:

```
server.js
```
Controller menangani request dari frontend lalu memberikan response kembali ke client.
Route berfungsi mengatur perpindahan halaman pada aplikasi React sehingga user dapat berpindah antar halaman tanpa reload website.

### Contoh Controller :
```
app.post('/api/login', (req, res) => {
    const { email, password } = req.body;

    if(email && password){
        return res.json({
            success: true,
            message: 'Login berhasil'
        });
    }

    return res.status(400).json({
        success: false,
        message: 'Email atau password salah'
    });
});
```
Controller bertugas menerima request dari user, memproses data input, lalu mengirim response berupa JSON ke frontend.

Service

Bagian Service berada di:
```
backend/api/authApi.js
```
Service digunakan untuk menangani komunikasi API menggunakan fetch.

#### Contoh Service :
```
export async function login(email, password) {
  return request('/login', {
    method: 'POST',
    body: JSON.stringify({ email, password }),
  });
}
```

Service berfungsi memisahkan logika request API dari tampilan sehingga kode menjadi lebih modular dan mudah digunakan ulang.
