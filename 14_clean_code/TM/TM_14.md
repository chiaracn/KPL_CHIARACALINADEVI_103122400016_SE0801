# Pendahuluan

Pada pengembangan perangkat lunak, kode yang baik tidak hanya harus berjalan dengan benar tetapi juga harus mudah dibaca, dipahami, dan dipelihara. Salah satu masalah yang sering ditemukan adalah penggunaan nested conditional (if bersarang) yang terlalu dalam sehingga alur program menjadi sulit dipahami.

Pada studi kasus ini diberikan dua implementasi fungsi `processUser()`. Tugasnya adalah menentukan kode mana yang lebih baik dan menjelaskan alasan pemilihannya dari sudut pandang Clean Code.

---

# Kode Pertama

```javascript
function processUser(user) {
    if (user) {
        if (user.isActive) {
            if (user.hasPermission) {
                return doSomething(user);
            }
        }
    }

    return null;
}
```

---

# Kode Kedua

```javascript
function processUser(user) {
    if (!isValidCandidate(user)) return null;

    return doSomething(user);
}

function isValidCandidate(user) {
    return user &&
           user.isActive &&
           user.hasPermission;
}
```

# Alasan Memilih Kode Kedua

## 1. Lebih Mudah Dibaca

Pada kode pertama terdapat tiga tingkat nested if.

```text
if (user)
 └── if (user.isActive)
      └── if (user.hasPermission)
```

Developer harus membaca setiap blok satu per satu untuk memahami alurnya.

Sedangkan pada kode kedua:

```javascript
if (!isValidCandidate(user))
    return null;
```

Alur program dapat langsung dipahami dalam satu kali baca.

---

## 2. Menggunakan Guard Clause

Kode kedua menerapkan teknik Guard Clause.

```javascript
if (!isValidCandidate(user))
    return null;
```

Guard Clause adalah teknik yang digunakan untuk menangani kondisi gagal di awal sehingga logika utama program tidak perlu berada di dalam banyak blok if.

Keuntungan:

* Mengurangi tingkat indentasi.
* Membuat kode lebih bersih.
* Mengurangi kompleksitas.

---

## 3. Lebih Sesuai Prinsip Clean Code

Menurut Clean Code oleh Robert C. Martin, fungsi sebaiknya:

* Pendek.
* Mudah dibaca.
* Memiliki satu tujuan yang jelas.

Kode kedua lebih memenuhi ketiga prinsip tersebut dibandingkan kode pertama.

---

## 4. Mudah Diperbaiki Saat Terjadi Bug

Bayangkan pukul 01.00 dini hari terdapat laporan bahwa pengguna tertentu tidak dapat mengakses sistem.

Pada kode pertama saya harus memeriksa:

```javascript
if (user)
if (user.isActive)
if (user.hasPermission)
```

satu per satu.

Sedangkan pada kode kedua saya cukup fokus pada:

```javascript
isValidCandidate(user)
```

Karena seluruh validasi sudah dipusatkan pada satu fungsi.

Hal ini mempercepat proses debugging dan mengurangi risiko kesalahan.

---

## 5. Lebih Mudah Dikembangkan

Misalnya perusahaan ingin menambahkan syarat baru:

```javascript
user.emailVerified
```

Pada kode pertama:

```javascript
if (user) {
    if (user.isActive) {
        if (user.hasPermission) {
            if (user.emailVerified) {
                ...
            }
        }
    }
}
```

Nested semakin dalam.

Sedangkan pada kode kedua cukup menambahkan satu kondisi:

```javascript
function isValidCandidate(user) {
    return user &&
           user.isActive &&
           user.hasPermission &&
           user.emailVerified;
}
```

Perubahan lebih sederhana dan aman.

# Refactoring yang Dilakukan

Transformasi dari:

```javascript
if (user) {
    if (user.isActive) {
        if (user.hasPermission) {
            return doSomething(user);
        }
    }
}
```

menjadi:

```javascript
if (!isValidCandidate(user))
    return null;

return doSomething(user);
```

Teknik yang digunakan:

* Extract Method
* Guard Clause
* Reduce Nesting
* Improve Readability
